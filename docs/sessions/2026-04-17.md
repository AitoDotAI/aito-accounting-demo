## Aito quirks discovered

- `_predict` returns values in `feature` key, not in a field-named key. `select` only supports `$p`, `feature`, `$why` — using the field name causes "field not found" error.
- `_relate` does not accept `select` — always returns full statistical breakdown.
- `_match` requires schema `link` properties on foreign key columns to traverse tables. Without links, it returns matched field values but not full rows.
- `_match` $p values are spread across all records in the target table, so even strong matches have low absolute $p (~0.19 for 230 records). Thresholds need calibration per table size.
- `load_dotenv(override=True)` needed because shell.nix sets default env vars that override .env.

## Decisions made

- uv over pip for dependency management (faster, lockfile, `uv run` avoids venv activation)
- Ports in 8200 range to avoid conflicts with concurrent dev work
- Anomaly score = 1 - max($p), with GL mismatch as separate signal boosted to gl_p * 0.8
- Rule mining uses single-field conditions only (vendor, category, vendor_country) — multi-field combinations deferred
